# Stroberi Release

## Overview

Use this skill for release operations that touch `app.json`, `package.json`, `eas.json`, build numbers, runtime version, or EAS build/submit flow.

## Release Checklist

1. Review `RELEASE_GUIDE.md`.
2. Bump release version intentionally:
   - `app.json` -> `expo.version`
   - `package.json` -> `version`
3. Validate release safety:
   - `yarn lint`
   - `yarn check:types`
4. Build via EAS (`production` profile) and verify outputs.
5. Commit any local version/build metadata changes generated by EAS.

## Guardrails

- Do not run release version bumps for regular feature work.
- Keep `runtimeVersion` compatibility in mind when deciding between store builds and OTA updates.
- Follow project channels in `eas.json` (`development`, `preview`, `production`).
